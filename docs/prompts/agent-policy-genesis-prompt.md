## Product vision

Build `agent-policy` as a serious open-source tool that provides:

1. A canonical machine-readable source of truth:
   - `agent-policy.yaml`

2. Renderers/generators for repository instruction files such as:
   - `AGENTS.md`
   - `CLAUDE.md`
   - `.github/copilot-instructions.md`

3. Validation and enforcement for commit attribution:
   - required footer syntax:
     `Assisted-by: AGENT_NAME:MODEL_VERSION`

4. Optional commit policy support for:
   - checkpoint commits during task execution
   - final squashed Conventional Commit for a completed task

5. Good OSS hygiene:
   - polished README
   - docs site or docs folder
   - examples
   - tests
   - schema docs
   - contribution guide
   - changelog/release notes starter
   - license
   - issue/pr templates
   - CI
   - install instructions

6. Claude Code marketplace/plugin-facing support under the name `agent47`
   - package whatever Claude Code metadata/config/integration is appropriate
   - if an actual marketplace manifest format is not available or not applicable, create a best-effort integration package and docs under `integrations/claude-code/agent47`
   - document assumptions clearly
   - do not invent fake official APIs without marking them as provisional

7. GitHub push readiness
   - prepare repository structure and git history cleanly
   - generate a final checklist for me to push to GitHub
   - if git remote is configured and pushing is possible in the environment, prepare the exact commands
   - if not possible, still make the repo fully ready and tell me the exact push commands at the end

### Step 4: Implement schema and config model
Design a clean config model, versioned from day one.

Suggested config:

```yaml
version: 1

project:
  name: my-repo

task_policy:
  commit_on_task_completion: true
  allow_checkpoint_commits: true
  require_user_prompt_for_squash: true

checkpoint_commit:
  enabled: true
  subject_format: "checkpoint(task={task_id}, seq={seq}, agent={agent}, model={model}, ts={ts}): {summary}"
  run_precommit: false

final_commit:
  conventional_commits: true
  run_precommit: true
  required_footers:
    - "Assisted-by"

attribution:
  assisted_by:
    required: true
    format: "Assisted-by: {agent}:{model}"

renderers:
  agents_md:
    enabled: true
    path: "AGENTS.md"
  claude_md:
    enabled: true
    path: "CLAUDE.md"
  copilot_instructions:
    enabled: true
    path: ".github/copilot-instructions.md"
```

You may refine this, but keep it simple and extensible.

### Step 6: Hook enforcement
Implement `install-hooks` so that it installs a working `commit-msg` hook in a sensible way.

Handle common cases carefully:
- `.git/hooks`

Hook validation must at least enforce:

Suggested regex:
`(?m)^Assisted-by:\s+[A-Za-z0-9._-]+:[A-Za-z0-9._-]+$`

Optionally support config-driven stricter checks.
